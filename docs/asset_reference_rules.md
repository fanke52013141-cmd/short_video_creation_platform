# Asset Reference Rules

本文件定义图片资产、分镜图和视频提示词之间的引用规则。所有项目都应遵守，除非用户明确要求某个项目临时偏离。

## 视频提示词中的 @ 规则

逐镜头视频提示词中的 `@` 资产分为必选、条件必选和条件可选。

| 类型 | 规则 |
|---|---|
| 分镜图 `@SHOT` | 每个 shot 必须声明。 |
| 人物 `@CHAR` | 主要人物必须声明，且应指向三视图人物资产。 |
| 音色 `@AUDIO` | 有台词、旁白、录音留言或可听见人声时必须声明。 |
| 场景 `@ENV` | 仅当镜头运动需要分镜图之外的空间信息时声明。 |
| 道具 `@PROP` | 默认不声明，只写入正文画面描述。 |

默认不在视频提示词中 `@` 的资产：

- 道具资产：`PROP_001`、`PROP_014` 等只写入画面描述，不作为 `@PROP` 声明。

原因：视频模型多参考时，道具图容易和人物/场景/分镜图抢权重，导致人物漂移、场景变形或镜头主体错位。

## 分镜图角色

分镜图属于镜头级参考，不属于道具。

| 角色 | 用法 |
|---|---|
| 首帧参考 | 锁定视频开始时的构图和主体初始状态。 |
| 尾帧参考 | 锁定视频结束时的构图和最终停留画面。 |
| 关键帧参考 | 锁定中段构图、光线、动作方向或情绪峰值。 |

如果外部平台只支持一种图片参考，优先上传当前镜头的分镜图，再上传人物和场景。

## 场景资产条件引用

分镜图通常已经包含场景信息，因此 `@ENV` 不是默认必选。

需要 `@ENV`：

- 分镜图是近景，但镜头要后拉到中景或全景。
- 镜头有明显摇、移、跟、升降，扩展出分镜图外空间。
- 人物从一个空间位置移动到另一个空间位置。
- 视频需要展示门口、楼道、街巷、礼堂、教室等完整空间。
- 分镜图只锁人物关系，无法提供足够场景连续性。

不需要 `@ENV`：

- 固定镜头。
- 只在分镜图构图内轻微推近。
- 特写脸部、手部或道具，背景不拓展。
- 分镜图本身已经覆盖完整空间。

每条视频提示词必须写明 `@ENV` 使用或不使用的理由。

## 音色资产条件引用

有台词、旁白、录音留言或可听见人声时，必须 `@AUDIO`。

没有人声时，不 `@AUDIO`。

如果缺少音色参考，不能把该 shot 标记为最终可生产，应向用户索要音频或标记占位。

详见 `docs/audio_reference_protocol.md`。

## 人物三视图

主要人物资产必须是三视图资产或明确映射到三视图资产。

详见 `docs/character_three_view_protocol.md`。

## 道具处理

道具资产仍然需要生成图片，尤其是母题道具和文字道具。道具图用于：

- 给用户确认美术风格和年代准确性。
- 给提示词编写提供视觉参考。
- 必要时作为外部平台的人工参考。
- 后期修图、叠字或剪辑包装参考。

但在逐镜头视频提示词中，道具通常写成自然语言画面描述，例如：

```text
桌上有三副碗筷，其中一副完全没人动。
空椅子上放着父亲的警帽。
警号数字 539527 在胸前短暂反光。
```

## 文字类画面

文字类画面仍然必须生成，不因为文字可能不稳定而跳过。

处理原则：

- 图片提示词应明确要求生成具体文字。
- 如果模型无法稳定生成，保留清晰构图和可修正区域。
- 视频提示词可以写“尽量呈现指定文字；如数字或文字不稳定，后期叠加”。
- 不能因为后期叠字困难而放弃生成文字画面。

适用对象：

- 作文本。
- 纸条。
- 家长会通知。
- 警号。
- 证件、票据、信件、贺卡。

## 一致性检查

视频提示词生成完成后至少检查：

- 每个镜头有建议时长。
- 每个镜头有对应 `@SHOT_XXX_STORYBOARD`。
- 每个镜头的主要 `@CHAR` 能在资产映射表找到，且具备三视图状态。
- 有人声的镜头有对应 `@AUDIO`。
- `@ENV` 的出现或缺席有引用决策说明。
- 默认情况下 `@PROP` 数量为 0。
- 中文提示词存在，英文提示词默认不存在。
