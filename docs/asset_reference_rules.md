# Asset Reference Rules

本文件定义当前 7 阶段主流程中，资产图、分镜图和视频提示词之间的引用规则。

## 命名原则

当前流程不使用 `CHAR_001`、`ENV_001`、`PROP_001`、`SHOT_001` 作为主引用名。

- 人物资产：使用稳定人物名或 `人物稳定名_持续变体`，例如 `林小满_雨夜居家装`。
- 场景资产：使用稳定具象场景名，例如 `雨夜客厅场景`。
- 道具资产：使用自然名称，例如 `手机`、`旧皮箱`。
- 分镜图：使用 `S###_分镜参考图`，例如 `S001_分镜参考图`。
- 视频段：使用 `V###`，例如 `V001`。

## 视频提示词中的 @ 规则

视频提示词必须先声明资产，再在正文中按完全一致的名称引用。

| 类型 | 当前规则 |
|---|---|
| 分镜图 | 每个 `V###` 必须声明其 `frame_references` 中的分镜图，例如 `@S001_分镜参考图`。 |
| 人物 | 主要人物必须声明为资产执行官固定的人物资产名，例如 `@林小满_雨夜居家装`。 |
| 场景 | 需要空间约束时声明稳定场景资产名，例如 `@雨夜客厅场景`。 |
| 道具 | 默认不写 `@PROP`，而是在正文中自然描述。 |
| 外部素材 | 仅在具体项目明确引入外部图片、视频或音频时使用，并需在提示词正文中说明用途。 |

## 分镜图角色

分镜图属于镜头级参考，不属于道具。

| 角色 | 用法 |
|---|---|
| `first_frame` | 锁定视频开始时的构图和主体初始状态。 |
| `last_frame` | 锁定视频结束时的构图和最终停留画面。 |
| `keyframe` | 锁定中段构图、光线、动作方向或情绪峰值。 |

角色先由 `video_segment_plan.json.segments[].frame_plan` 锁定，分镜提示词和视频提示词必须保持一致。

## 场景资产引用

分镜图通常已经包含场景信息，因此场景资产不是每条提示词都必须重新强调全貌。

需要声明或强调场景资产的情况：

- 分镜图是近景，但镜头要后拉到中景或全景。
- 镜头有明显摇、移、跟、升降，扩展出分镜图外空间。
- 人物从一个空间位置移动到另一个空间位置。
- 视频需要展示门口、楼道、街巷、礼堂、教室等完整空间。
- 分镜图只锁人物关系，无法提供足够场景连续性。

不需要额外强化场景资产的情况：

- 固定镜头。
- 只在分镜图构图内轻微推近。
- 特写脸部、手部或道具，背景不拓展。
- 分镜图本身已经覆盖完整空间。

无论是否强化场景，正文引用的场景名称必须与资产声明区一致。

## 道具处理

道具默认写入正文画面描述，不使用 `@PROP`。

示例：

```text
桌上有三副碗筷，其中一副完全没人动。
空椅子上放着父亲的警帽。
警号数字 539527 在胸前短暂反光。
```

只有在某个项目明确需要独立道具图时，`asset_executor` 才应将该道具标记为 `generation_required=true`，并为其写入 `output_prompt_path`。即便道具图已生成，视频提示词默认仍以正文描述控制，不强制 `@PROP`。

## 文字类画面

文字类画面仍然必须生成或明确处理，不因为文字可能不稳定而跳过。

处理原则：

- 图片提示词应明确要求生成具体文字。
- 如果模型无法稳定生成，保留清晰构图和可修正区域。
- 视频提示词可以写“尽量呈现指定文字；如数字或文字不稳定，后期叠加”。
- 如果视频阶段写了文字生成要求，不要再同时写“无字幕 / 无文字”这类相冲突约束。

适用对象：

- 作文本。
- 纸条。
- 家长会通知。
- 警号。
- 证件、票据、信件、贺卡。

## 一致性检查

视频提示词生成完成后至少检查：

- 每个 `V###` 有 `task_type=pipeline_shot_generation`。
- 每个 `V###` 有 `source_shots`，且 source shots 连续、有序、同一 `scene_id`。
- 每个 `V###` 有 `frame_references`，且覆盖当前全部 source shots。
- 每个 `V###` 的分镜图声明与 `frame_references` 一致。
- 每个 `V###` 的人物、场景资产名与 `asset_manifest.json` 和 `shot_asset_map.json` 一致。
- 默认情况下不出现 `@PROP`。
- 中文提示词存在，英文提示词默认不存在。
- 结尾包含无字幕、无 Logo、无水印约束。
