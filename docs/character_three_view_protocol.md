# Character Three-View Protocol

人物资产必须支持视频生成中的身份稳定。单张人物肖像不足以稳定人物在动作、转身、跟拍和多镜头连续中的一致性。

## 基本规则

每个主要人物的每个状态变体都必须生成三视图：

- 正视图：front
- 侧视图：side
- 后视图：back

示例：

```text
CHAR_003_A_儿子童年/
├── front.png
├── side.png
└── back.png
```

如果工具更适合单图，也可以生成一张三视图设定图：

```text
char_003_a_son_child_three_view.png
```

但提示词必须明确要求“三视图设定图”，不能只生成单人肖像。

## 适用范围

必须三视图：

- 主角。
- 有多镜头连续出现的人物。
- 有动作、转身、走动、侧身、背影的角色。
- 需要跨年龄或状态保持一致的角色。

可简化为单图：

- 只出现一次的背景人物。
- 远景或虚化人物。
- 不需要身份连续性的群众。

## 状态变体

人物不同年龄、服装、职业状态、情绪阶段必须拆开：

```text
CHAR_003_A_儿子童年
CHAR_003_B_儿子少年
CHAR_003_C_儿子成年缉毒警
CHAR_003_D_儿子成年父亲
```

每个变体都需要自己的三视图，不能把多个年龄阶段放进同一张最终资产图。

## 提示词要求

人物提示词必须包含：

- 正、侧、后三个视角。
- 同一角色、同一服装、同一年龄、同一发型、同一体态。
- 中性站姿，无遮挡。
- 低表情强度，便于后续视频驱动。
- 年代、职业、地域服装准确性。

## 视频提示词引用

视频提示词中的人物 `@CHAR` 应对应三视图资产或三视图映射，而不是单张肖像。

示例：

```text
@CHAR_003_B_儿子少年（三视图人物资产）
```

如果当前项目只有单图人物资产，应在 `production_status.csv` 中标记：

```text
character_view_status=single_view_needs_upgrade
```

此状态允许草稿测试，不建议用于最终批量视频生成。
