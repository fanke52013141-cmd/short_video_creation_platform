# Character Identity Quad Protocol

人物资产必须支持视频生成中的身份稳定。单张人物肖像不足以稳定人物在动作、转身、跟拍和多镜头连续中的一致性。

## 基本规则

每个主要人物及其必要的持续变体生成一张 2x2 身份四宫格：

- 面部近景：锁定五官、发型和识别细节。
- 全身正面：锁定体型、比例和完整服装。
- 全身三分之四侧面：锁定立体轮廓和侧脸。
- 全身背面：锁定背部发型、服装和轮廓。

示例：

```text
林小满_雨夜居家装.png
```

四格必须属于同一个人、同一年龄、同一发型、同一体型和同一套服装。人物标准身份格不使用俯视图。

## 适用范围

必须身份四宫格：

- 主角。
- 有多镜头连续出现的人物。
- 有动作、转身、走动、侧身、背影的角色。
- 需要跨年龄或状态保持一致的角色。

可简化为单图：

- 只出现一次的背景人物。
- 远景或虚化人物。
- 不需要身份连续性的群众。

## 持续变体

人物仅在年龄、持续服装/发型、持续伤痕或身份转变时拆开：

```text
儿子_童年
儿子_少年校服
儿子_成年缉毒警
儿子_受伤后
```

临时哭泣、愤怒、接电话、走路和镜头景别不拆资产。名称禁止添加“状态”二字。

## 提示词要求

人物提示词必须包含：

- 面部近景、全身正面、全身三分之四侧面、全身背面。
- 同一角色、同一服装、同一年龄、同一发型、同一体态。
- 中性站姿，无遮挡。
- 低表情强度，便于后续视频驱动。
- 年代、职业、地域服装准确性。

## 视频提示词引用

视频提示词中的人物引用应对应批准的四宫格身份资产，而不是任意单张肖像。

示例：

```text
@儿子_少年校服（四宫格人物身份资产）
```

如果当前项目只有单图人物资产，应在 `production_status.csv` 中标记：

```text
character_view_status=single_view_needs_upgrade
```

此状态允许草稿测试，不建议用于最终批量视频生成。
