# Video Prompt Loop Protocol

逐镜头视频提示词必须循环生成，不能只一次性生成总文件。

## 输出结构

```text
outputs/05_video_prompts/
├── shots/
│   ├── SHOT_001.md
│   ├── SHOT_002.md
│   └── ...
├── shot_video_prompt_index.md
├── shot_video_prompts.md
└── video_prompt_asset_reference.md
```

用户最终主要查看 `shot_video_prompts.md`，但生成过程必须留下 `shots/SHOT_XXX.md`，用于证明每条镜头已独立生成、独立自检、可单独返修。

## 循环流程

对 `storyboard.json.shots` 中每个镜头依次执行：

1. 读取当前 shot 的叙事功能、时长、景别、镜头运动、台词和资产 ID。
2. 读取当前 shot 对应分镜图，判断其角色：首帧参考、尾帧参考或关键帧参考。
3. 读取当前 shot 的主要人物资产；人物必须使用三视图资产或明确的三视图映射。
4. 判断是否有台词、旁白、录音留言或可听见的人声。
5. 有人声时，检查对应 `@AUDIO` 音色参考；缺失则停止该 shot 并向用户索要音频，或写入 `missing` 占位并标记不可最终生产。
6. 根据镜头运动判断是否需要 `@ENV` 场景参考。
7. 生成单条中文视频提示词，保存到 `outputs/05_video_prompts/shots/SHOT_XXX.md`。
8. 对单条提示词自检，不合格则重写该 shot。
9. 全部 shot 通过后，按镜头顺序汇总到 `shot_video_prompts.md`。

## 资产声明决策

每条 `SHOT_XXX.md` 必须包含：

```text
【引用决策】
- 分镜图：@SHOT_XXX_STORYBOARD，首帧参考 / 尾帧参考 / 关键帧参考
- 人物：@CHAR_...
- 场景：使用 / 不使用；说明原因
- 音色：使用 / 不使用；说明原因
- 道具：不 @，写入正文
```

## 场景 @ENV 条件

`@ENV` 不是默认必选。分镜图通常已经包含场景信息。

需要 `@ENV` 的情况：

- 分镜图是近景，但镜头要后拉到中景或全景。
- 镜头有横移、摇镜、跟拍、升降或明显空间拓展。
- 人物从一个空间位置移动到另一个位置。
- 镜头要展示分镜图之外的门口、楼道、街巷、礼堂、教室全貌。
- 分镜图只锁人物关系，无法提供足够空间连续性。

不需要 `@ENV` 的情况：

- 固定镜头。
- 只在分镜图构图内轻微推近。
- 特写脸部、手部或道具，背景不拓展。
- 分镜图本身已经覆盖完整空间。

无论是否 `@ENV`，都必须在引用决策中解释。

## 音色 @AUDIO 条件

有台词、旁白、录音留言或可听见人声时，必须 `@AUDIO`。

没有人声的镜头不 `@AUDIO`。

如果缺少音色参考，不允许假装完成最终版；必须：

- 向用户索要音频；或
- 使用文字音色占位并标记 `audio_reference_status=missing`。

## 道具规则

视频提示词默认不 `@PROP`。

道具必须写入正文，例如：

```text
桌上三副碗筷中有一副完全没人动。
空椅子上放着父亲的警帽。
警号 539527 在胸前短暂反光。
```

## 自检

每条单 shot 文件必须检查：

- 是否只有中文提示词。
- 是否有建议时长。
- 是否声明 `@SHOT_XXX_STORYBOARD`。
- 是否声明主要人物。
- 是否对 `@ENV` 使用或不使用给出理由。
- 有人声时是否声明 `@AUDIO`。
- 是否没有 `@PROP`。
- 是否写了镜头运动和时间节奏，而不只是静态画面描述。
